2 ## Motivation
3 
4 This module provides an Argument Parser that does not require registration.
5 It uses a weak type and casts depending to the context and thus removes manual casting.
6 It's further possible to merge different Argument Parsers with different policies.
7 
8 ## Syntax
9 
10 The following syntax is legal for this argument parser.
11 
12 #### free arguments
13 
14 aka positional arguments / unnamed arguments
15 * any combination of characters as long as they don't start with `-`
16  and do not contain `=`
17 
18 #### named arguments
19 
20 _Note_: `val` has the same constrains as the free arguments
21 * `key=val` note that there is no white-space between the `key`, `=` and `val`
22 * `key=-val` one way to set negative values
23 * `--key val`
24 * `--key=val`, `--key=-val`
25 * `-k val` if the key `k` only has one character
26 * `-k=val`, `-k=-val`
27 * `-kval`, `-k-val` alternative notation for `k=val`, `k=-val`
28 
29 #### flags
30 * `--flag`
31 * `-f` if `f` is only one character long
32 
33 #### remarks
34 
35 * a valid command line is build by the above syntax, separated by white-space, i.e:\n
36 `free0 free1 --flag -f key=10`
37 * it is not possible to have negative values as free arguments
38 * it is not allowed to have a flag and named argument with the same name,
39  since there is no registration.
